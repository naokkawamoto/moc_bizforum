更新（2026年4月8日）: ルート index.html の SYSTEM USER 導線（user_login.html 経由）、管理画面ログイン、共通ヘッダー・フッター、user_settings_staff.html、user_activity_log.html、user_visitor_detail.html の参加・不参加表示等を現行モックに合わせて反映。

対象デバイス: デスクトップ・タブレット（表示幅 1024px 以上を想定。例：iPad Pro 1024×1366px）

対応ブラウザ: Google Chrome、Safari、Microsoft Edge の各ブラウザについて、リリースから1年以内のバージョンを対象とする（例：2026年2月時点では各ブラウザの直近1年分のメジャーバージョン）。

注意: 受付アプリ（スマートフォン向け）の要件定義は user_site/requirements_specification.html を参照してください。

1. user_project_list.html - イベントリスト

15. user_login.html - 管理画面ログイン

項目内容
概要ルート index.html の「SYSTEM USER（管理画面）」から遷移。メールアドレス・パスワード・パスワードリセット・ログインボタンを配置。ログイン成功時のモックは user_project_list.html へ遷移（認証処理なし）。
パスワードリセットリンク操作でモーダル表示。「パスワードがリセットされました。新しいパスワードを設定してください。」「ログインに戻る」で閉じるダミー。
その他「モックアップ入口に戻る」で index へ。ヘッダーにシステム名（管理画面）。

16. user_activity_log.html - アクティビティログ

項目内容
概要操作履歴の閲覧・監査用。ヘッダー「設定」→「アクティビティログ」から遷移。検索条件：開始・終了日時、名前、ページ名・URL（部分一致）。一覧は日時・名前・ページ名とURL（2行表示）。1ページ最大50件、ページネーション。モックはダミーデータ約20件。

共通仕様

項目 内容
対象デバイス デスクトップ・タブレット（表示幅 1024px 以上。例：iPad Pro 1024×1366px）
対応ブラウザ Google Chrome、Safari、Microsoft Edge の各ブラウザについて、リリースから1年以内のバージョンを対象とする。
システム名 イベント入退場管理システム（受付アプリは user_site を参照）
レイアウト イベントリスト：左カラム（イベント名リスト）＋中央メイン＋右カラム（お知らせ・VIP来場通知）。イベントトップ・来場者・メール・名札・来場者・入退室：左サイドメニュー（イベントトップ／来場者・メール・名札／来場者・入退室。VIP来場のお知らせはサイドメニューに含めない）＋中央メイン。
現行ファイル 本要件定義に記載の画面（目次16項＋共通ヘッダー・フッター）＋ user_login.html。削除済みファイル（report_detail, schedule, overview, drawing, budget, staff, partner, other, work_type, partner設定, chat, user_detail）は対象外。ルート index.html から管理画面へは必ず user_login.html を挟む（2026年4月）。
