イベント入退場管理システム 要件定義書（system_user_cms）

作成日: 2026年2月17日

対象デバイス: デスクトップ・タブレット（表示幅 1024px 以上を想定。例：iPad Pro 1024×1366px）

対応ブラウザ: Google Chrome、Safari、Microsoft Edge の各ブラウザについて、リリースから1年以内のバージョンを対象とする（例：2026年2月時点では各ブラウザの直近1年分のメジャーバージョン）。

注意: 受付アプリ（スマートフォン向け）の要件定義は user_site/requirements_specification.html を参照してください。

1. user_project_list.html - イベントリスト

項目 内容
ページタイトル イベント入退場管理
概要 登録済みのイベントを一覧表示するメインページ。パネル/リスト切り替え、ステータスフィルタリング（準備中/受付中/終了）機能あり。

レイアウト構成

エリア 幅 機能・要件
左カラム 300px イベント名リスト（リンク一覧）→ クリックで詳細ページへ遷移
中央カラム 可変 メインコンテンツ（イベントカード/テーブル表示）
右カラム 300px お知らせ・VIP来場通知タブ

機能詳細

新規イベント作成ボタン 大きく目立つ赤いボタン（btn-danger）。クリックで新規イベント登録ページへ遷移 → user_project_create.html
ステータスタブ
フィルタータブ ステータス別にフィルタリング すべて / 準備中 / 受付中 / 終了

パネル/リスト切替 ボタンでカード表示⇔テーブル表示を切り替え -

イベントカード カードクリックで詳細ページへ遷移 イベント名 / VIP来場アイコン・バッジ / ステータスバッジ / イベント画像 / 開催日時 / 開催場所 / 担当者 / 登録者数 / 入室実績

イベントテーブル 行クリックで詳細ページへ遷移 状況 / イベント名（VIP来場アイコン付き） / VIP来場（バッジ表示） / 開催日時 / 開催形式 / 担当者 / 登録者数 / 入場済

共通仕様

項目 内容
対象デバイス デスクトップ・タブレット（表示幅 1024px 以上。例：iPad Pro 1024×1366px）
対応ブラウザ Google Chrome、Safari、Microsoft Edge の各ブラウザについて、リリースから1年以内のバージョンを対象とする。
