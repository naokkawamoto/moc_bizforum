イベント入退場受付アプリ（user_site）要件定義書

作成日: 2026年2月17日

更新（2026年4月8日）: ルート index.html の構成（USER SITE 導線・業務フロー）、フッターメニュー（利用規約・プライバシーポリシーを含む）の現行モックを反映。管理画面（system_user_cms）側の変更は system_user_cms/requirements_specification.html を参照。

更新（2026年4月9日）: イベントトップ（受付）のヘッダー統合（「受付／VIP管理アプリ」・VIP来場ベル・プログラム・ログアウト）、site_read_keynote.html のヘッダー文言（「受付VIP管理アプリ」・戻る）、プログラム概要の統計から入室予定を削除（入室・退室実績のみ）を反映。

対象デバイス: iPhone 16以降、Android（スマートフォン）。表示幅は393px基準で最適化。

対応OS: iOS はリリースから2年以内のバージョン（例：2026年時点では iOS 18 以降）、Android はリリースから2年以内のOS（例：2026年時点では Android 14 以降）を対象とする。

注意: 入口は index.html の「USER SITE（受付、VIP管理）」から login.html へ遷移。ログイン後、受付アプリ（site_project_list.html）または VIP管理アプリ（site_project_list_vip.html）に分岐。受付アプリにはVIP来場ページへの遷移はなく、VIP管理アプリには受付QRリーダーはない。同一 index に「SYSTEM USER（管理画面）」は system_user_cms/user_login.html 経由で管理画面へ（2026年4月）。

0. login.html - ログイン（受付・VIP管理）

項目内容
ページタイトルログイン - 受付アプリ / VIP管理アプリ
概要index の USER SITE（受付、VIP管理）から遷移する中間ログインページ。スマートフォン表示（幅393px基準）。タブは使わず、縦に「受付アプリ ログイン」「VIP管理アプリ ログイン」のカードを並べ、それぞれ メールアドレスのみ（パスワードなし）とログインボタン（バリデーションなしで押下時に遷移）。
